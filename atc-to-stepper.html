Accessibility notes

  • ✓

    +/− buttons: full tab stops: Both buttons are in the tab order with descriptive accessible names. Arrow keys on the input provide a second equivalent path — not a replacement.

  • ✓

    Remove button at qty = 1: "Remove" is semantically distinct from "decrement". Full tab stop, accessible name "Remove Blue Running Shoes from cart", per eBay's "with delete button" pattern.

  • ✓

    Two-stage ATC announcement: "Adding to cart" fires immediately. "[Product] added to cart. 1 item. Quantity stepper now available." fires 150 ms after the DOM restructure settles.

  • ✓

    Announce without clear-then-set: Directly sets new content, toggling a trailing NBSP so identical consecutive messages still diff in the DOM.

  • ✓

    Focus management: Focus moves to the number input when the stepper appears. Focus returns to the ATC button when quantity hits 0.

  • ✓

    ATC loading state (WCAG SC 4.1.2): The button's accessible name switches to "Adding [Product] to cart" during loading, paired with aria-busy. SR users know their action was received.

  • ✓

    No form landmark: The stepper is not wrapped in a <form> element, avoiding spurious "Form Start / Form End" announcements.
